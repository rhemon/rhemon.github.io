Zultan Website

Description

Zultan provides a platform where buyers and sellers connect with each other online. The target is to onboard all local shops of Bangladesh and let the people of Bangladesh explore all shops and markets virtually while staying at home. I was the backend developer of this platform and worked alongside a frontend developer to complete this website.

THe website includes 4 different panels, one for the general customers, one for the vendors/shop owners, one for the partners/agents that take care of delveries and onboard vendors and the last is the admin panel where we can check all transactions and activities.

Tools Used

React, Django, HTML & CSS, Boostrap, JavaScript, Heroku, AWS S3, PrivateEmail, PostgreSQL, Celery, Papertrail

Problems Faced

  • Allowing POST requests to Django from React
  • Two people working on same files resulted to conflicts in work and would make it harder to merge the work.
  • Heroku refreshes its storage and so media files couldn't be uploaded there.

How did I deal with the problems?

We used axios to do post and get requests to the django server, and post requests would fail because of csrf token was missing, hence to resolve the issue I used the getCookie function provided in the Django documentation and set that as a default X-CSRFToken header for axios. To deal with conflicts and to improve the management in our team work we decide keep track of all task by making use of GitHub issues. This way we could track what each other's currently working on and reduce down the chances of causing merge conflicts. For uploading media files, unlike digitalocean we could not use it to store on the server provided by heroku as it would refresh it storages and only keep the files tracked by the git repository, therefore we started to use AWS S3 to store and upload all media files.
